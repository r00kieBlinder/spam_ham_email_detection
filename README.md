# Spam Email Detection System

A machine learning-based text classification model developed for the Artificial Intelligence Laboratory (CSE 3812) to automatically distinguish between legitimate communications ("ham") and spam.

**Author:** F. Abrar

## Overview

The volume of unsolicited bulk emails poses significant challenges, often serving as a delivery mechanism for malware and phishing attacks. This project leverages Natural Language Processing (NLP) techniques for feature extraction and compares the performance of probabilistic and linear classifiers to implement an accurate filtering solution. 

## Dataset

The model utilizes a pre-labeled dataset (`spam_ham_dataset.csv`) encapsulating historical email data with diverse writing styles and subjects.

* **Total Samples:** The dataset contains 5,171 emails.
* **Target Variable:** It uses a binary indicator where `0` signifies 'Ham' and `1` signifies 'Spam'.
* **Features:** The input relies on raw text content from the body and subject of the email.

## Methodology

The data processing and training pipeline consists of the following technical steps:

* **Text Vectorization:** Raw textual data is transformed into a continuous statistical weight using the Term Frequency-Inverse Document Frequency (TF-IDF) algorithm via `scikit-learn`.
* **Stop-Words Filtering:** Standard English stop-words are explicitly filtered out to enhance the signal-to-noise ratio.
* **Dimensionality:** The vectorization process generated a massive high-dimensional matrix consisting of 50,140 unique vocabulary features.
* **Data Splitting:** The dataset is partitioned into an 80/20 split (4,136 training samples and 1,035 testing samples) using a static random state (`random_state=42`) for reproducibility.

## Models Evaluated

Two distinct classification algorithms were tested on the high-dimensional feature space:

1. **Multinomial Naive Bayes:** A probabilistic classifier applying Bayes' theorem under the assumption of conditional independence.
2. **Logistic Regression:** A linear model computing binary outcomes by optimizing a decision boundary across the space.

## Results

Both models demonstrated strong predictive capabilities, but Logistic Regression substantially outperformed Naive Bayes. This discrepancy occurred because the continuous weightings generated by TF-IDF perfectly complemented the mathematical optimization process of the Logistic Regression model, whereas Naive Bayes is fundamentally designed for integer feature counts.

| Model | Testing Accuracy |
| :--- | :--- |
| Multinomial Naive Bayes | 92.17% |
| Logistic Regression (max_iter=1000) | 98.84% |

## Getting Started

To explore the classification pipeline locally:

1. Clone this repository to your local machine.
2. Ensure you have Python installed along with `pandas` and `scikit-learn`.
3. Place the `spam_ham_dataset.csv` file in the same directory as the notebook.
4. Execute the Jupyter Notebook `Spam_Email_Detection.ipynb` to run the data processing, model training, and view sample predictions.
