# Spam Email Detection System

[cite_start]A machine learning-based text classification model developed for the Artificial Intelligence Laboratory (CSE 3812) to automatically distinguish between legitimate communications ("ham") and spam[cite: 5, 11].

**Author:** F. Abrar

---

## Overview

[cite_start]The volume of unsolicited bulk emails poses significant challenges, often serving as a delivery mechanism for malware and phishing attacks[cite: 9, 10]. [cite_start]This project leverages Natural Language Processing (NLP) techniques for feature extraction and compares the performance of probabilistic and linear classifiers to implement an accurate filtering solution[cite: 12]. 

## Dataset

[cite_start]The model utilizes a pre-labeled dataset (`spam_ham_dataset.csv`) encapsulating historical email data with diverse writing styles and subjects[cite: 22, 23].

* [cite_start]**Total Samples:** The dataset contains 5,171 emails[cite: 25].
* [cite_start]**Target Variable:** It uses a binary indicator where `0` signifies 'Ham' and `1` signifies 'Spam'[cite: 27].
* [cite_start]**Features:** The input relies on raw text content from the body and subject of the email[cite: 26].

## Methodology

The data processing and training pipeline consists of the following technical steps:

* [cite_start]**Text Vectorization:** Raw textual data is transformed into a continuous statistical weight using the Term Frequency-Inverse Document Frequency (TF-IDF) algorithm via `scikit-learn`[cite: 31, 32].
* [cite_start]**Stop-Words Filtering:** Standard English stop-words are explicitly filtered out to enhance the signal-to-noise ratio[cite: 34].
* [cite_start]**Dimensionality:** The vectorization process generated a massive high-dimensional matrix consisting of 50,140 unique vocabulary features[cite: 35].
* [cite_start]**Data Splitting:** The dataset is partitioned into an 80/20 split (4,136 training samples and 1,035 testing samples) using a static random state (`random_state=42`) for reproducibility[cite: 37, 38, 39, 40].

## Models Evaluated

[cite_start]Two distinct classification algorithms were tested on the high-dimensional feature space[cite: 42]:

1. [cite_start]**Multinomial Naive Bayes:** A probabilistic classifier applying Bayes' theorem under the assumption of conditional independence[cite: 43, 44].
2. [cite_start]**Logistic Regression:** A linear model computing binary outcomes by optimizing a decision boundary across the space[cite: 45, 46].

## Results

[cite_start]Both models demonstrated strong predictive capabilities, but Logistic Regression substantially outperformed Naive Bayes[cite: 51]. [cite_start]This discrepancy occurred because the continuous weightings generated by TF-IDF perfectly complemented the mathematical optimization process of the Logistic Regression model, whereas Naive Bayes is fundamentally designed for integer feature counts[cite: 53, 56].

| Model | Testing Accuracy |
| :--- | :--- |
| Multinomial Naive Bayes | [cite_start]92.17% [cite: 49] |
| Logistic Regression (max_iter=1000) | [cite_start]98.84% [cite: 49] |
